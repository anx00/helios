# 2026-03-03 — Ensemble Counting + Estrategia Micro Value

## Contexto
Analisis del documento `docs/analysis/Profesional_weather_trading_analysis.md.md` contra el estado actual de Helios. Se identifico que el GAP critico era la falta de ensemble counting real (Helios usaba pronostico determinista, no los 82 miembros de ECMWF+GFS). Con $50 de bankroll, la unica estrategia viable es micro-apuestas estilo gopfan2.

## Archivos nuevos

### `collector/ensemble_fetcher.py`
- Fetcher completo para Open-Meteo Ensemble API
- Obtiene 51 miembros ECMWF (`ecmwf_ifs025`) + 31 miembros GFS (`gfs_seamless`) = 82 total
- `fetch_ensemble_snapshot(station_id, target_date)` — distribuciones por bucket
- `fetch_ensemble_edge(station_id, market_buckets)` — comparacion ensemble vs precios de mercado
- Cache de 10 minutos (modelos se actualizan cada 6h)
- Soporte para mercados en °F (US) y °C (EU)

## Archivos modificados

### `core/papertrader.py`
- Nueva estrategia **micro_value_ensemble** (5ta estrategia, color violeta #a855f7)
- Metodo `_select_micro_value_candidate()`: escanea todos los buckets, compara ensemble_prob vs market_price
- Pre-fetch de ensemble data en `run_station_once()` antes del loop de estrategias
- Budget fijo de $1 por trade (bypassa Kelly para esta estrategia)
- Minimo 8% de edge para entrar
- Soporta tanto BUY YES como BUY NO

### `web_server.py`
- `GET /api/ensemble/{station_id}?target_date=YYYY-MM-DD` — datos raw del ensemble (members, stats, buckets)
- `GET /api/ensemble/edge/{station_id}?min_edge=8` — edges detectados vs mercado actual
